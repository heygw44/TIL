# 스택

## 스택의 정의와 성질
- 정의: 한쪽 끝에서만 원소를 넣거나 뺄 수 잇는 자료구조.
- 별명: FILO - 선입후출. 프링글스 통이나 엘리베이터를 생각해보자.
- 핵심 성질
  - 원소의 추가(`push`)가 $O(1)$이다.
  - 원소의 제거(`pop`)가 $O(1)$이다.
  - 제일 상단의 원소 확인(`top`)가 $O(1)$이다.
  - 제일 상단이 아닌 나머지 원소들의 확인/변경은 원칙적으로 불가능하다.

## 자료구조 구현

### STL `std::stack`
- 헤더 `<stack>
- 코테에서는 특별한 이유가 있는게 아니라면 STL을 사용하자.
- 주요함수: `pop`, `push`, `top`, `empty`, `size`
- 주의사항: 스택이 비어있을 때 `top`이나 `pop`를 호출하면 런타임 에러가 발생하므로 항상 `empty`여부를 먼저 확인하자.

### 배열을 이용한 구현(직접 구현)
```cpp
const int MX = 1000005;
int dat[MX];
int pos = 0; // 다음에 원소가 들어갈 인덱스 (= 현재 원소의 개수)

void push(int x) { dat[pos++] = x; }
void pop() { pos--; }
int top() { return dat[pos-1]; }
```

## 스택의 활용 사례
1. 수식의 괄호 쌍
   - 올바른 괄호 문자열인지 판단할 때 사용.
   - 여는 괄호는 push, 닫는 괄호는 pop하여 짝 맞추기.
2. 전위/후위 표기법 변환 및 계산
   - 연산자 우선순위 처리에 사용.
3. 웹 브라우저 '뒤로 가기'
   - 가장 최근에 방문한 페이지가 먼저 나와야하므로 스택 구조가 적합.

## 심화 테크닉: 모노톤 스택
코테에서 스택을 활용하는 가장 중요한 유형 중 하나이다.
- 문제상황: 왼쪽(혹은 오른쪽)에서 나보다 큰 첫 번째 건물을 찾자.
- 핵심 아이디어
  - 스택안의 원소들을 내림차순 혹은 오름차순으로 유지.
  - 새로운 원소가 들어올 때, 스택의 `top`이 조건을 만족하지 않으면(예: 나보다 높이가 낮으면) 과감하게 버린다(`pop`).
  - 이유: 나(새 원소)보다 높이가 낮은 애들은 어차피 나한테 가려져서 뒤에 있는 원소에게 보이지 않는다.(쓸모없는 정보는 삭제하자.)
- 시간 복잡도: 모든원소는 스택에 한번 들어가고 한번 나오므로 $O(N)$에 확인 가능하다.
